# Дальнейшие действия

1. **VPS:** после `git pull` в `vitaly1/max-bot-vitaly` выполнить шаги из [AFTER_GIT_PULL.md](AFTER_GIT_PULL.md): обновить зависимости, перезапустить `max-bot`.
2. **Проверка:** убедиться, что один сервис `max-bot` (без v2) запущен и в логах нет ошибок: `journalctl -u max-bot -f`.
3. **Мониторинг:** при сбоях индексации смотреть логи; при необходимости добавить алерт (например, в Telegram) при повторных ошибках в `run_index_site`.
4. **Опционально:** проверить лимит длины поста в MAX API и при необходимости добавить обрезку в `publishers.py` (как для Telegram).
