# Контент-машина napitki133.ru — памятка для заказчика

Кратко: бот по расписанию (МСК) берёт материалы с сайта, генерирует текст через ИИ и публикует в **Telegram** и **MAX** (три тематических канала). Управление графиком и текстами — через **Google Таблицу**.

**Секреты (токены, ключи) в этот файл не вносятся** — они только на сервере в файле `.env`.

---

## 1. Сервер (заполняет исполнитель)

| Поле | Значение |
|------|----------|
| Хост (IP или домен) | `___________________________` |
| SSH (пользователь) | обычно `root` или отдельный пользователь под бота |
| Путь к проекту на сервере | обычно `~/vitaly1/max-bot-vitaly` |
| Сервис systemd | обычно `max-bot.service` |

**Проверка, что бот запущен (выполняет админ на сервере):**

```bash
sudo systemctl status max-bot
```

**Логи в реальном времени:**

```bash
sudo journalctl -u max-bot -f
```

**Логи за сегодня:**

```bash
sudo journalctl -u max-bot --since today
```

## 2. Google Таблица (заполняет исполнитель)

| Поле | Значение |
|------|----------|
| Название / назначение | панель управления ботом |
| Ссылка на таблицу | `https://docs.google.com/spreadsheets/d/________________/edit` |
| ID таблицы (фрагмент URL между `/d/` и `/edit`) | хранится на сервере в `V2_GOOGLE_SHEET_ID` |

**Доступ:** у сервисного аккаунта Google (email из JSON-ключа) должна быть роль **Редактор** на эту таблицу.

### Листы

| Лист | Зачем |
|------|--------|
| **Расписание** | Канал, тип поста, день недели (`mon`–`sun`), время (`ЧЧ:ММ`). Типы: `main`, `shop`, `cross`, `site`, `radio` (при необходимости — `rotation` для AV). |
| **Промпты** | Задания для ИИ по ключам (`main_travel`, `shop`, `cross` и т.д.). |
| **Маппинг** | Какие рубрики сайта относятся к какому каналу. |
| **Настройки** | Интервалы повторов, режим выборки и т.п. |
| **История** | Автоматический журнал публикаций (бот дописывает строки). |

**Первичная разметка таблицы:** в репозитории есть скрипт `init_sheets.gs` (Google Apps Script) — его вставляют в **Расширения → Apps Script** и запускают `initBotSheets()`.

## 3. Как менять расписание

1. Откройте лист **«Расписание»**.
2. Меняйте строки: канал (`travel` / `lifhaki` / `drinks`), тип, день, время.
3. Новые **валидные** строки бот подхватывает **без перезапуска**: планировщик перечитывает таблицу примерно **каждые 30 минут** (плюс кэш Google Sheets на стороне чтения ~10 минут — итоговая задержка до ~40 минут).
4. Если нужно применить сразу — попросите админа **перезапустить сервис** (см. раздел 4).

Ошибки в строках (пустой день, `*` вместо дня, нет канала для `main` и т.п.) бот **пропускает**; при полной «пустоте» после фильтрации может включиться резервное расписание из кода — лучше не оставлять таблицу пустой.

---

## 4. Обновление кода на сервере (делает исполнитель / админ)

Из каталога с репозиторием (пример):

```bash
cd ~/vitaly1/max-bot-vitaly
git pull origin main
```

Зависимости (если менялся `requirements.txt`):

```bash
source venv/bin/activate
pip install -r requirements.txt
deactivate
```

Перезапуск бота:

```bash
sudo systemctl restart max-bot
```

## 5. Что заказчик делает сам

- Редактирует **Расписание**, **Промпты**, при необходимости **Маппинг** и **Настройки** в Google Таблице.
- Смотрит **Историю** публикаций.

## 6. Что заказчик не трогает

- Файл `.env` на сервере, токены ботов, ключ OpenRouter, JSON ключ Google.
- Репозиторий и перезапуск сервиса — по согласованию с исполнителем.
